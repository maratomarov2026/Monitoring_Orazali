# PRD — MediaWatch AI: Система мониторинга упоминаний в сети интернет

**Версия документа:** 1.0  
**Дата:** 25 июня 2025  
**Статус:** Draft  
**Тип продукта:** Веб-приложение с ИИ-модулем  

---

## 1. Обзор продукта

### 1.1 Назначение

MediaWatch AI — интеллектуальная система мониторинга упоминаний в сети интернет, предназначенная для автоматического поиска, фиксации, классификации и анализа публикаций и комментариев по заданным темам на веб-сайтах, в социальных сетях, блогах и форумах.

### 1.2 Целевая аудитория

- Рядовые пользователи (обыватели), желающие отслеживать упоминания интересующих тем
- Сотрудники государственных органов
- Сотрудники квазигосударственных структур
- Пресс-службы и отделы коммуникаций

### 1.3 Цели продукта

- Автоматический поиск и фиксация упоминаний по заданной теме в интернете
- Анализ тональности найденных материалов (позитив / нейтральный / негатив)
- Мониторинг комментариев под публикациями на всех площадках
- Визуализация географии упоминаний по странам мира и по регионам Казахстана
- Формирование структурированных аналитических справок
- Автоматическая рассылка справок адресатам по каналам Telegram, WhatsApp и e-mail

---

## 2. Пользовательские сценарии

### 2.1 Единоразовый мониторинг

> Пользователь вводит тему, задаёт период, запускает однократный поиск, получает справку по итогам.

### 2.2 Постоянный мониторинг

> Пользователь настраивает расписание (2–10 раз в сутки), система автоматически запускает проверки в заданные интервалы времени и рассылает обновлённые справки адресатам.

### 2.3 Реагирование на негативные упоминания

> По итогам мониторинга система формирует рекомендации — пресс-служба оперативно реагирует на выявленный негатив на основании конкретных ссылок и предложенных действий.

## 3. Функциональные требования

### 3.1 Блок A — Поисковый запрос и период мониторинга

| # | Требование |
|---|-----------|
| A-1 | Поле ввода темы или ключевого слова для мониторинга |
| A-2 | Выбор временного периода (дата начала и дата окончания) |
| A-3 | Выбор источников: СМИ и порталы, социальные сети, комментарии, блоги и форумы, государственные ресурсы (.gov.kz) |
| A-4 | Выбор языков мониторинга: казахский, русский, английский, китайский |

### 3.2 Блок Б — Облако ключевых слов

| # | Требование |
|---|-----------|
| Б-1 | Перед началом мониторинга система автоматически генерирует облако ключевых слов на основе введённой темы |
| Б-2 | Облако включает варианты написания на 4 языках: казахском, русском, английском, китайском |
| Б-3 | Облако включает корректные написания и распространённые ошибочные варианты |
| Б-4 | Облако включает хэштеги, падежные формы, аббревиатуры и составные формы темы |
| Б-5 | Пользователь может добавлять новые слова в облако вручную |
| Б-6 | Пользователь может удалять нежелательные слова из облака |
| Б-7 | Итоговое облако сохраняется и применяется при каждом последующем запуске мониторинга |

### 3.3 Блок В — Настройка расписания

| # | Требование |
|---|-----------|
| В-1 | Режим «Единоразовый мониторинг» — однократный запуск за указанный период |
| В-2 | Режим «Постоянный мониторинг» — регулярные автоматические проверки |
| В-3 | В режиме постоянного мониторинга пользователь задаёт число проверок в сутки (минимум 2, максимум 10) |
| В-4 | Пользователь задаёт временной интервал: время первой и последней проверки |
| В-5 | Система равномерно распределяет проверки внутри заданного интервала |

### 3.4 Блок Г — Адресаты рассылки

| # | Требование |
|---|-----------|
| Г-1 | Возможность добавить неограниченное число адресатов |
| Г-2 | Поддерживаемые каналы доставки: Telegram (по @username), WhatsApp (по номеру телефона), электронная почта |
| Г-3 | К каждому адресату привязывается имя или должность |
| Г-4 | Адресаты могут быть удалены из списка |
| Г-5 | Справка отправляется всем адресатам одновременно по нажатию кнопки или автоматически по расписанию |

### 3.5 Блок Д — Справка по итогам мониторинга

Справка состоит из следующих разделов:

#### Раздел I. Общая статистика упоминаний

- Общее число зафиксированных упоминаний за период
- Разбивка по типу площадки: СМИ и порталы, социальные сети, блоги и прочее

#### Раздел II. Тональность публикаций

- Процентное и количественное соотношение: позитив / нейтральный / негатив
- Визуальная шкала тональности
- Разбивка тональности по типам площадок

#### Раздел III. Топ-10 публикаций по тональности

- Три подраздела: позитивные, нейтральные, негативные
- Для каждой публикации: заголовок, краткое содержание, источник, дата

#### Раздел IV. Подборка ссылок

- Полный список ссылок на все зафиксированные упоминания
- Для каждой ссылки: источник, заголовок, дата, тональность
- Возможность выгрузки полного списка в формате XLSX

#### Раздел V. Рекомендации по реагированию

- Конкретные рекомендуемые действия по нейтрализации негативных упоминаний
- Рекомендации по увеличению доли позитивных упоминаний

#### Раздел VI. Анализ комментариев *(отдельный раздел)*

- Общее число зафиксированных комментариев
- Разбивка по площадкам (Instagram, Facebook, Telegram, X, YouTube, блоги, форумы, комментарии под статьями СМИ)
- Тональность комментариев: процент и количество по каждой категории
- Разбивка тональности комментариев по площадкам
- Топ-50 комментариев с наибольшим числом реакций в формате:
  - Автор (имя / @username)
  - Площадка
  - Тональность
  - Текст комментария
  - Разбивка реакций: лайки, комментарии, репосты, сохранения, суммарный охват
- Фильтрация топ-50 по тональности
- Выгрузка топ-50 комментариев в формате XLSX

#### Раздел VII. Карта упоминаний по миру

- Пузырьковая визуальная карта с указанием стран
- Размер пузыря пропорционален числу упоминаний
- Цвет пузыря отражает преобладающую тональность
- Интерактивная подсказка при наведении: страна, число упоминаний, разбивка тональности
- Таблица с полным списком стран и данными по упоминаниям

#### Раздел VIII. Карта упоминаний по регионам Казахстана

- Схематическая карта с отображением всех 20 регионов:
  - 3 города республиканского значения: Астана, Алматы, Шымкент
  - 17 областей: Акмолинская, Актюбинская, Алматинская, Абайская, Атырауская, Восточно-Казахстанская, Жамбылская, Жетысуская, Западно-Казахстанская, Карагандинская, Костанайская, Кызылординская, Мангистауская, Павлодарская, Северо-Казахстанская, Туркестанская, Улытауская
- Города республиканского значения визуально выделены отдельным типом маркера
- Цвет заливки региона отражает преобладающую тональность
- Числовые пузырьки с числом упоминаний по каждому региону
- Интерактивная карточка при клике на регион: число упоминаний, тональность, ведущие площадки
- Таблица с сортировкой по охвату с фильтрами: все регионы / только города / только области

## 5. Навигационная структура интерфейса

```
MediaWatch AI
├── Настройка
│   ├── Новый мониторинг        ← ввод темы, периода, источников, языков
│   ├── Облако слов             ← генерация и редактирование ключевых слов
│   ├── Расписание              ← единоразовый / постоянный, частота
│   └── Адресаты                ← Telegram, WhatsApp, e-mail
└── Результаты
    ├── Справка-отчёт           ← разделы I–V + VII–VIII
    ├── Комментарии             ← раздел VI, топ-50
    ├── Карта упоминаний        ← мир и регионы Казахстана
    └── Мои задачи              ← запущенные и завершённые мониторинги
```

## 7. Требования к ИИ-модулю

| Функция | Описание |
|---------|---------|
| Генерация облака ключевых слов | По введённой теме ИИ автоматически формирует варианты написания на 4 языках, включая ошибочные |
| Классификация тональности | Каждая публикация и каждый комментарий классифицируется как позитивный, нейтральный или негативный |
| Краткое содержание | ИИ формирует аннотацию для каждой публикации, попавшей в топ-10 |
| Рекомендации | ИИ формирует конкретные рекомендации по реагированию на основе выявленных негативных упоминаний |
| Определение географии | ИИ определяет регион или страну источника публикации или комментария |

## 8. Форматы выгрузки

| Формат | Содержимое |
|--------|-----------|
| Просмотр в интерфейсе | Полная интерактивная справка со всеми разделами и картами |
| PDF | Справка в официальном формате для передачи руководству |
| XLSX — полный список упоминаний | Все зафиксированные публикации со ссылками, источниками и тональностью |
| XLSX — топ-50 комментариев | Автор, площадка, текст, разбивка реакций |
| Рассылка | Отправка справки по Telegram, WhatsApp, e-mail всем указанным адресатам |

## 9. Ограничения и допущения

- Система охватывает открытые (публичные) источники; закрытые группы и приватный контент не индексируются
- Точность определения тональности зависит от качества ИИ-модели и может требовать ручной корректировки в спорных случаях
- Частота мониторинга в постоянном режиме ограничена диапазоном 2–10 проверок в сутки
- Географическая карта Казахстана отображает схематическое расположение регионов; точные границы не являются картографическими

## 10. Критерии приёмки (Definition of Done)

- [ ] Пользователь может ввести тему, задать период и запустить мониторинг
- [ ] ИИ генерирует облако ключевых слов на 4 языках
- [ ] Облако доступно для редактирования до запуска
- [ ] Настройки расписания (единоразовый / постоянный, 2–10 раз в сутки) работают корректно
- [ ] Добавление адресатов по Telegram, WhatsApp и e-mail функционирует
- [ ] Справка формируется со всеми 8 разделами
- [ ] Раздел VI содержит топ-50 комментариев с разбивкой реакций и фильтрацией
- [ ] Карта мира отображает упоминания по странам с тональностью
- [ ] Карта Казахстана отображает все 20 регионов (17 областей + 3 города республиканского значения)
- [ ] Справка успешно отправляется адресатам по всем трём каналам
- [ ] Выгрузка в XLSX работает для списка упоминаний и топ-50 комментариев
